Bested.h

My C, single-header library that I use in every C/C++ project I make

the C programming language, in its base form, is one of the worst programming languages to program in. This single .h file is ~3,000 lines of code. and makes programming in C much nicer, it includes things like:
  • Typedef'ing the uint64_t family of types to u64, this one change makes it so much nicer to work in c
    • Just something as simple as this creates much better looking, (and more readable), code 
      // ===================================================
//                    Classic Ints
// ===================================================

typedef uint64_t        u64;
typedef uint32_t        u32;
typedef uint16_t        u16;
typedef uint8_t         u8;

typedef int64_t         s64;
typedef int32_t         s32;
typedef int16_t         s16;
typedef int8_t          s8;
    • There is no circumstance where you would want your int type to change based on platform.
      • Way more problems are solved by just not using int, that it creates.
        • If you have a performance issue, I promise you its not because your integer is the wrong size for the register.
  • Arena Allocator & Arena_Pool for `Memory Management`
    • 50% of all `Memory Management` issues can be solved with these 2 data structures

    // (local_persist is just static, but with a name that makes sense)
    local_persist u8 buffer[1 << 16] = ZEROED;
    Arena_Add_Buffer_As_Storage_Space(&a, buffer, sizeof(buffer));

    // maybe you don't want any more space than what you just gave it?
    a.panic_when_trying_to_allocate_new_page = true;
}

  • Dynamic & Type Safe Generic Array's
    • 90% of the remaining memory management issues can be solved by putting a Arena allocator onto an Array 
      • // yes, this is valid C
//
// this is an array of signed 64-bit numbers!
Array(s64) array = {};

// typedef's allow you to pass these array into other functions!
//
// (you cant pass a Array(Foo) directly, fundamental problem with C)
//
// (also Array(Type) is a macro, macro's are cool.)
typedef Array(Foo) Foo_Array;

// now you never have to free this array!
//
// also there is some special code that makes growing the
// array faster if certain conditions are met.
array.allocator = &arena;

// you can also use a pool,
// just remember to give it back later.
array.allocator = Pool_Get();

{ // the array interface
    // access an element
    array.items[i];

    // get the count
    array.count;

    Array_Append(&array, item);
    Array_Insert(&array, index, item);

    Array_Reserve(&array, num_to_reserve);

    // remove and move.
    Array_Remove(&array, index)
    // the cooler remove
    Array_Swap_And_Remove(&array, 3);

    // iteration helper.
    Array_For_Each(it, &array) {
        // item index.
        u64 index = it - array.items;
        // remember `it` is a pointer to a s64
        printf("%zu: %ld\n", index, *it);
    }
    // or do it yourself.
    for (u64 i = 0; i < array.count; i++) {
        s64 *item = array.items[i];
    }
}
